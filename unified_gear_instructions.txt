## Library Selection Strategy

### BOSL2 Library (PREFERRED - Modern Approach)

**Advantages:**
- English parameter names (mod, teeth, thickness vs. modul, zahnzahl, breite)
- Precise center distance calculation with `gear_dist()`
- Built-in profile shifting and undercutting prevention
- Comprehensive gear types with consistent interface
- Superior 3D printing optimization

**Usage Pattern:**
```openscad
include <BOSL2/std.scad>
include <BOSL2/gears.scad>

// Quality settings - ALWAYS include
$fa = 1; $fs = 0.4;

// example spur gear
spur_gear(circ_pitch=5, teeth=20, thickness=8, shaft_diam=5);
```

**Core Functions:**
- `spur_gear()` - Standard external gears
- `ring_gear()` - Internal/annular gears  
- `bevel_gear()` - Conical gears for angle drives
- `rack()` - Linear gear rack
- `gear_dist()` - Calculate center distance for meshing gears
- `auto_profile_shift()` - Automatic profile shift calculation

BOSL2's `gear_dist()` function provides precise center distance calculations for proper gear meshing.

External Gear Positioning 
```openscad
// Example: Two meshing gears with proper rotation
include <BOSL2/std.scad>
include <BOSL2/gears.scad>
$fn=32;
teeth1 = 20;
teeth2 = 40;
circ_pitch = 5;
thickness = 10;

// Calculate center distance
d12 = gear_dist(circ_pitch=circ_pitch, teeth1=teeth1, teeth2=teeth2);

// Position and rotate gears for proper meshing
color("blue") 
    zrot($t * 360 / teeth1) 
        spur_gear(circ_pitch=circ_pitch, teeth=teeth1, thickness=thickness, shaft_diam=5);

color("green")
    translate([d12, 0, 0])
        zrot(-$t * 360 / teeth2 + 180/teeth2)
            spur_gear(circ_pitch=circ_pitch, teeth=teeth2, thickness=thickness, shaft_diam=8);

// ROTATION ANALYSIS:
// Blue gear (20 teeth): rotates +18° per $t increment (360/20 = 18°) 
// Green gear (40 teeth): rotates -9° + 4.5° offset per $t (360/40 = 9°, 180/40 = 4.5°)
// - Negative rotation ensures opposite direction meshing
// - Phase offset (180/teeth2) prevents tooth-to-tooth collision  
// - Result: teeth mesh into gaps, not into other teeth
```

Helical Gears on Non-Parallel Axes
```openscad
// Example: Helical gears on skew axes with proper positioning
include <BOSL2/std.scad>
include <BOSL2/gears.scad>
ang1 = 30;
ang2 = 10;
circ_pitch = 5;
n = 20;
dist = gear_dist_skew(
   circ_pitch=circ_pitch,
   teeth1=n, teeth2=n,
   helical1=ang1, helical2=ang2);
left(dist/2) spur_gear(
       circ_pitch, teeth=n, thickness=10,
       shaft_diam=5, helical=ang1, slices=12,
       gear_spin=-90
   );
right(dist/2)
xrot(ang1+ang2)
spur_gear(
       circ_pitch=circ_pitch, teeth=n, thickness=10,
       shaft_diam=5, helical=ang2, slices=12,
       gear_spin=90-180/n
   );
```

## Gear Rotation Fundamentals - CRITICAL FOR PREVENTING TOOTH OVERLAP

### Why Gear Rotation is Essential

**CRITICAL**: Positioning gears with only `translate()` will result in overlapping teeth! Gears must be rotated to the correct phase relationship for proper meshing.

### Basic Rotation Formula

All meshing gears must rotate proportionally based on their tooth count:
```openscad
// Basic rotation calculation
rotation_angle = $t * 360 / number_of_teeth;
```

Where:
- `$t` = OpenSCAD animation time variable (0 to 1)
- `360` = full rotation in degrees
- `number_of_teeth` = tooth count of the gear

### Meshing Gear Rotation Relationships

For two meshing gears, their rotations must be synchronized:
```openscad
// Driving gear (positive rotation)
a1 = $t * 360 / teeth1;

// Driven gear (negative rotation with phase offset)
a2 = -$t * 360 / teeth2 + 180/teeth2;
```

**Key principles:**
- **Driving gear**: Positive rotation (`+`)
- **Driven gear**: Negative rotation (`-`) to mesh in opposite direction
- **Phase offset**: `180/teeth` aligns teeth for proper engagement
- **Ratio relationship**: Smaller gears rotate faster (more degrees per `$t`)

### Phase Offset Calculation

The phase offset `180/teeth` ensures teeth engage properly:
- `180/teeth` = half a tooth pitch in degrees
- Aligns teeth so they mesh rather than collide
- Different for each gear based on its tooth count

### Advanced Rotation: gear_spin Parameter

BOSL2 provides `gear_spin` for fine-tuning rotation:
```openscad
spur_gear(circ_pitch=5, teeth=20, thickness=10, gear_spin=90);
```

Use `gear_spin` to:
- Fine-tune tooth alignment
- Adjust meshing engagement
- Optimize tooth contact patterns

### Multiple Gear Assemblies

For complex assemblies with multiple gears, each gear needs its own rotation calculation:
```openscad
// From the working example:
a1 =  $t * 360 / n1;                    // Driving gear
a2 = -$t * 360 / n2 + 180/n2;          // First driven gear
a3 = -$t * 360 / n3 - 3*90/n3;         // Second driven gear  
a4 = -$t * 360 / n4 - 3.5*180/n4;      // Third driven gear
```

Each calculation includes specific phase adjustments for optimal meshing.

# Gear Meshing and Rotation Guide

## Critical Concept: Proper Gear Meshing

When positioning gears to mesh, you must consider **rotation offset** to prevent tooth-to-tooth collision. Gears need to be positioned so that teeth mesh into gaps, not into other teeth.

## The Half-Tooth Rule

For proper meshing, the second gear typically needs to be rotated by **half a tooth pitch**:
- Rotation angle = `180 / number_of_teeth` degrees
- This ensures tooth-to-gap alignment

## MCAD Gear Meshing

MCAD gears **ALWAYS** require manual rotation for proper meshing:

### Two Spur Gears
```openscad
use <MCAD/involute_gears.scad>

teeth1 = 20;
teeth2 = 30;
cp = 5;

// Calculate center distance
center_distance = (teeth1 + teeth2) * cp / (2 * PI);

// First gear (no rotation)
gear(number_of_teeth = teeth1, circular_pitch = cp, rim_thickness = 8);

// Second gear (with rotation offset)
translate([center_distance, 0, 0])
    rotate([0, 0, 180/teeth2])  // CRITICAL: Half tooth rotation
    gear(number_of_teeth = teeth2, circular_pitch = cp, rim_thickness = 8);
```

### Rack and Pinion
```openscad
use <MCAD/involute_gears.scad>

pinion_teeth = 15;
cp = 5;
pitch_radius = (pinion_teeth * cp) / (2 * PI);

// Rack
rack(number_of_teeth = 20, circular_pitch = cp, thickness = 10);

// Pinion (needs rotation for meshing)
translate([20, pitch_radius, 5])
    rotate([0, 0, 360/pinion_teeth/2])  // Half tooth rotation
    gear(number_of_teeth = pinion_teeth, circular_pitch = cp, rim_thickness = 10);
```

## BOSL2 Gear Meshing

BOSL2 has **better default positioning** but rotation can still be added for:
- Fine-tuning meshing
- Animation purposes
- Specific tooth alignment requirements

### Two Spur Gears (BOSL2)
```openscad
include <BOSL2/std.scad>
include <BOSL2/gears.scad>

// BOSL2 often meshes correctly without rotation
center_dist = gear_dist(circ_pitch=5, teeth1=20, teeth2=30);

spur_gear(circ_pitch=5, teeth=20, thickness=8);

translate([center_dist, 0, 0])
    // Optional: rotate([0, 0, 180/30])  // Can add for fine-tuning
    spur_gear(circ_pitch=5, teeth=30, thickness=8);
```

### Alternative: Using gear_spin Parameter
BOSL2 provides a `gear_spin` parameter for rotation:
```openscad
spur_gear(
    circ_pitch = 5,
    teeth = 30,
    thickness = 8,
    gear_spin = 180/30  // Built-in rotation parameter
);
```

## Key Differences

| Aspect | MCAD | BOSL2 |
|--------|------|-------|
| Default meshing | Poor - always needs rotation | Good - often works without |
| Rotation method | `rotate([0, 0, angle])` | `rotate()` or `gear_spin` parameter |
| Rack positioning | Position at pitch radius | Use gear_dist() helper |
| Required rotation | Always | Optional but recommended |

## Visual Verification

Always render from multiple angles to verify proper meshing:

### Check for Issues:
1. **Tooth collision**: Teeth hitting teeth (needs rotation)
2. **Gap too large**: Excessive backlash (positioning error)
3. **Partial overlap**: Wrong center distance calculation

### Good Meshing Signs:
1. Teeth fit cleanly into gaps
2. Consistent contact along tooth face
3. No visual interference
4. Smooth appearance of mesh line

## Animation Testing

Use OpenSCAD's animation to verify meshing:
```openscad
// Animate to check meshing
angle1 = $t * 360;
angle2 = -$t * 360 * (teeth1/teeth2);  // Opposite direction, gear ratio

rotate([0, 0, angle1])
    gear(number_of_teeth = teeth1, ...);

translate([center_distance, 0, 0])
    rotate([0, 0, angle2 + 180/teeth2])  // Include mesh offset
    gear(number_of_teeth = teeth2, ...);
```

## Common Pitfalls

1. **Forgetting rotation offset**: Results in tooth-to-tooth collision
2. **Wrong rotation direction**: Use same sign for offset as gear rotation
3. **Rack positioning errors**: Position at pitch radius, not pitch + addendum
4. **Inconsistent libraries**: MCAD and BOSL2 have different defaults

## Best Practices

1. **Always include rotation for MCAD gears**
2. **Test with multiple view angles**
3. **Use animation to verify smooth meshing**
4. **Document rotation calculations in comments**
5. **Consider backlash for real-world printing** (add 0.1-0.2mm)

## Formula Reference

### Center Distance
```
center_distance = (teeth1 + teeth2) * circular_pitch / (2 * π)
```

### Pitch Radius
```
pitch_radius = teeth * circular_pitch / (2 * π)
```

### Rotation Offset (half tooth)
```
rotation_angle = 180 / teeth  // in degrees
```

### Full Tooth Rotation
```
tooth_angle = 360 / teeth  // in degrees
```

# BOSL2 Gear Rotation Notes

## BOSL2 vs MCAD Rotation Requirements

### Key Difference
- **MCAD**: Rotation is **MANDATORY** for proper meshing
- **BOSL2**: Rotation is **OPTIONAL** (better default positioning)

### When to Add Rotation in BOSL2

While BOSL2 often meshes correctly without explicit rotation, you may want to add it for:

1. **Fine-tuning meshing** - Optimize tooth engagement
2. **Animation** - Ensure smooth rotation animation
3. **Specific requirements** - Match exact tooth alignment needs
4. **Consistency** - When mixing with MCAD components

### BOSL2 Rotation Methods

#### Method 1: Using rotate()
```openscad
translate([center_dist, 0, 0])
    rotate([0, 0, 180/teeth2])  // Optional rotation
    spur_gear(circ_pitch=5, teeth=teeth2, thickness=8);
```

#### Method 2: Using gear_spin parameter
```openscad
translate([center_dist, 0, 0])
    spur_gear(
        circ_pitch = 5,
        teeth = teeth2,
        thickness = 8,
        gear_spin = 180/teeth2  // Built-in rotation
    );
```

## Updated BOSL2 Examples with Optional Rotation

### L2.2 (BOSL2 Meshing Gears) - Enhanced
```openscad
include <BOSL2/std.scad>
include <BOSL2/gears.scad>
$fa = 1; $fs = 0.4; $fn = 64;

circ_pitch = 4.71;
teeth1 = 12;
teeth2 = 36;

center_dist = gear_dist(circ_pitch=circ_pitch, teeth1=teeth1, teeth2=teeth2);

// First gear
spur_gear(circ_pitch=circ_pitch, teeth=teeth1, thickness=8, shaft_diam=4);

// Second gear with optional rotation for perfect meshing
translate([center_dist, 0, 0])
    spur_gear(
        circ_pitch = circ_pitch,
        teeth = teeth2,
        thickness = 8,
        shaft_diam = 6,
        gear_spin = 180/teeth2  // Optional but recommended
    );
```

### L3.3 (BOSL2 Rack and Pinion) - Enhanced
```openscad
include <BOSL2/std.scad>
include <BOSL2/gears.scad>
$fa = 1; $fs = 0.4; $fn = 64;

circ_pitch = 5;
pinion_teeth = 16;
rack_length = 80;

rack_teeth = round(rack_length * PI / circ_pitch);

// Rack
rack(
    pitch = circ_pitch,
    teeth = rack_teeth,
    thickness = 6,
    backing = 4,
    width = 8
);

// Pinion with optional rotation for better meshing
pitch_radius = (pinion_teeth * circ_pitch) / (2 * PI);
translate([0, pitch_radius, 0])
    spur_gear(
        circ_pitch = circ_pitch,
        teeth = pinion_teeth,
        thickness = 8,
        shaft_diam = 4,
        gear_spin = 360/pinion_teeth/2  // Optional half-tooth rotation
    );
```
